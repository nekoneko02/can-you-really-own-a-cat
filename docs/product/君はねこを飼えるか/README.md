# 君はねこを飼えるか？

## プロダクト概要

- プロダクト名：君はねこを飼えるか？
- コンセプト：「ねこを飼うか迷っている人に、判断材料を与え、ゲームで疑似体験させ、より適切な判断を促す」
- アプローチ：責任感を押し付けるのではなく、**「気づき」を与える**ことを重視
  - 文字や写真だけでは理解しにくい日常の具体的な場面を疑似体験させる
  - 「○○という場面で、ねこがxxxという状況になる」という気づきを提供
  - 最終的な飼育判断はプレイヤー自身に委ねる

## ミッション・ビジョン・タグライン

- **ミッション（会社レベル）**：すべての動物が幸せな世界をつくる
- **ミッション（プロダクトレベル）**：猫を飼う前に適切な知識を学び、責任ある飼育を促進する
- **ビジョン**：飼育シミュレーションを通じて、猫と人の適切な関係性を構築できる社会を実現する
- **タグライン**：「飼う前に、猫を知ろう」

## ゲームで疑似体験

### 基本方針：「責任感」ではなく「気づき」を提供

ゲームで責任感を強制することは困難です。代わりに以下のアプローチを採用します：

#### 1. 具体的な場面の体験

- **日常の大変さを具体的に体験させる**
  - 夜中に鳴き続ける猫（睡眠不足の体験）
  - 突然の嘔吐・病院代のシミュレーション（想定外の出費）
  - 引っかき傷・破壊行動（家具への被害）
  - トイレ掃除の頻度（日常の手間）

#### 2. 気づきを促す仕組み

- 体験後に「この状況、実際に続けられますか？」と問いかけ
- プレイログを元にした「飼育適性診断」レポート
- 「こういう場面が苦手だった」という振り返り機能

#### 3. 最終判断はユーザーに委ねる

- ゲーム側は判断を強制せず、「気づき」の材料を提供
- 「飼うのをやめる」という選択も成功として扱う
- プレイ後アンケートで「気づいたこと」を自由記述

#### 4. リアルさと楽しさのバランス

- **楽しさ**: かわいい猫との触れ合い、なつき度の向上
- **リアルさ**: 予期せぬトラブル、日常の手間、費用の可視化
- ゲーム進行で徐々にリアルな場面を増やし、ギャップを体感させる

#### 5. 「誤った自信」への対策

- プレイ後レポートで「実際はもっと大変」を明示
- 「ゲームで飼えた ≠ 実際に飼える」をユーザーに伝える
- ゲームはあくまで「判断材料」であり「保証」ではないことを明記

## 成果目標（初年度）

### 定量目標

- **ダウンロード数**：10～50人
  - 想定流入：身内・技術ブログ読者など限定的なリーチ
  - 収益化は優先せず、フィードバック収集とプロダクト改善を重視
- **シナリオ完了率**：60%
  - 実際にゲームを最後までプレイしたユーザーの割合
- **行動変容率**：50%
  - プレイ前後のアンケートで「飼育に対する意識が変化した」と回答した割合
- **ユーザー満足度**：80%
  - 「気づきを得られた」「判断材料として役立った」という評価の割合

### 定性目標

- **深い気づきを得られたか**: ユーザーからの具体的なフィードバックで検証
- **ゲームの完成度**: バグが少なく、体験の質が高いこと
- **シナリオの網羅性**: 判断材料の整理で定義した優先度の高い体験をカバーできているか

## 成果測定方法

実際の飼育行動や放棄率は追跡不可能なため、アプリ内アンケートで行動変容を測定します。

### 測定方針

- **開始時アンケート**: 猫を飼いたい度合い、飼育に対する不安や理解度を測定
- **終了時アンケート**: プレイ後の飼いたい度合い、気づきの有無、判断材料としての有用性を測定
- **行動変容の測定**: 開始時と終了時のアンケート比較で、意識の変化を捉える
  - 例: 「とても飼いたい → 迷っている」（慎重になった）
  - 例: 「迷っている → とても飼いたい」（覚悟が決まった）
  - いずれも成功として評価
