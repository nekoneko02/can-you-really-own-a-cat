# 機能一覧

## 概要

本ドキュメントでは、「君はねこを飼えるか？」の機能を一覧化し、コンセプト達成のための網羅性と実現可能性を検証します。

---

## 機能分類の方針

### 基本方針

- **判断材料の整理.mdに基づいた優先順位づけ**
- **MVPは1週間～1ヶ月の飼育体験に絞る**（レビュー事項6-1で決定済み）
- **A×Y領域（事前学習で防げる×基準が曖昧）に注力**

### 優先度の定義

- 🔴 **MVP必須**：初期リリースで必ず実装する機能
- 🟠 **MVP推奨**：初期リリースで実装したい機能（工数次第で調整）
- 🟡 **将来バージョン**：価値検証後に追加する機能
- ⚪ **検討中**：実装要否を検討する機能

## 1. オンボーディング機能

### 1-1. プレイ前情報入力 🔴 MVP必須

**目的**: ユーザーの初期状態と期待値を測定

**機能内容**:

プロフィール設定画面で自然に入力できる形式で実装（1-3と統合）

- 「猫を飼いたいと思っていますか？」
  - 非常に飼いたい / やや飼いたい / 迷っている / あまり飼いたくない
- 「飼育に対する不安はありますか？」（複数選択可）
  - 時間的負担 / 経済的負担 / 住環境 / 睡眠への影響 / 衛生面 / 生活の自由度 / その他
- 「どの程度、猫の飼育について調べましたか？」
  - 全く調べていない / 少し調べた / かなり調べた / 専門的に学んだ

**データ活用**:

- 終了時アンケートと比較し、行動変容を測定
- ユーザータイプ（A/B/Cタイプ）の判定

**成功基準**:

- 回答率: 90%以上
- 完答時間: 1分以内

**決定事項**:

- アンケート形式ではなく、プロフィール設定の一部として実装
- 必須項目を絞り、離脱率を抑える

---

### 1-2. ゲーム説明・期待値設定 🔴 MVP必須

**目的**: ユーザーに「気づき」を得るツールであることを明示

**機能内容**:

- オープニング画面で以下を伝える：
  - このゲームは「飼えるか/飼えないか」を判断するためのツール
  - 責任感を押し付けるのではなく、「気づき」を与える（レビュー事項2-1）
  - 楽しさ + リアルな大変さを体験できる
  - 「やっぱりやめた」も成功として扱う
- 統計データの表示（最小限）
  - 令和5年度の猫の殺処分数: 6,899頭のみ表示
  - 詳細データは終了時レポート（4-2）で提示

**成功基準**:

- スキップ率: 30%未満（多くのユーザーが説明を読む）
- プレイ後アンケートで「目的を理解していた」: 80%以上

**決定事項**:

- オープニングでは殺処分数のみ表示（離脱率を抑える）
- 詳細な統計データは終了時レポート（4-2）で提示
- MVP期: テキストベース / 将来: グラフ化

---

### 1-3. 基本情報チェックリスト 🔴 MVP必須

**目的**: A×X領域（基準が明確）の情報提供

**機能内容**:

- プロフィール設定画面で自然に入力できる形式で実装
- 経済的負担の確認
  - 自由に使えるお金/月（入力: 数値）
  - 初期費用2〜3万円の説明
- 時間的余裕の確認
  - 自由時間/日（入力: 数値）
  - 朝夕最低30分、トイレ掃除、ブラッシングの説明
- 住環境の確認
  - 住環境（選択肢: 持ち家ペット可 / 賃貸ペット可 / 賃貸ペット不可 / その他）
  - ペット可物件か？部屋の広さは十分か？
- バリデーション
  - 警告レベル: 基準未満でも警告を出してゲーム続行可能
  - エラーレベル: 賃貸ペット不可の場合は警告強め

**成功基準**:

- チェックリスト完了率: 70%以上

**決定事項**:

- プロフィール設定画面で自然に入力できる形式に変更
- チェックリスト形式ではなく、通常の入力フォームとして実装
- 工数: +4時間

## 2. コアゲームプレイ機能

### 2-1. 飼育シミュレーション（基本ループ） 🔴 MVP必須

**目的**: 日常のお世話を体験させる

**機能内容**:

- **1日のサイクル**（朝・昼・夕・夜の4フェーズ）
  - 朝: 食事準備、トイレ掃除
  - 昼: 仕事中（猫の様子を確認できない）
  - 夕: 食事準備、遊び、トイレ掃除
  - 夜: 睡眠（夜中に起こされるイベント）

- **基本的なお世話**
  - 食事やり（1日2回）
  - トイレ掃除（1日2〜3回）
  - 水の交換（1日1回）
  - 遊び（なつき度に影響）

- **状態管理**
  - 猫の健康度（0〜100）
  - 満腹度（0〜100）
  - なつき度（0〜100）
  - ストレス度（0〜100）
  - プレイヤーの体力（0〜100、睡眠で回復）

- **時間経過**
  - 1週間〜1ヶ月の体験（MVP範囲）
  - 時間スキップ機能（「1日進める」ボタン）

**成功基準**:

- シナリオ完了率: 60%以上（レビュー事項6-1）
- 平均プレイ時間: 30分〜1時間

**決定事項**:

- MVP期はターン制（シナリオに応じた適切な時間単位）でシンプルに実装
- ゲーム性は最小限に抑え、体験の質を重視
- 育成ゲーム要素は避ける（楽しさよりリアルさを優先）

---

### 2-2. A×Y領域の体験イベント 🔴 MVP必須

**目的**: 「本当に我慢できる？」を疑似体験で問う

**決定事項**:

- イベント発生頻度: ネガティブ:ポジティブ = 6:4
- ポジティブイベントを最低3個追加（2-5-5参照）

#### 2-2-1. 日常のお世話の継続性

- 忙しい朝のトイレ掃除イベント
  - 選択肢: 「今日はサボろう」「遅刻覚悟で掃除する」
  - 影響: 猫のストレス度↑、部屋の臭い↑
- 疲れた夜の食事準備イベント
  - 選択肢: 「手抜きでドライフードだけ」「ウェットフードも用意」
  - 影響: 猫の満足度、なつき度

#### 2-2-2. 睡眠への影響

- 夜中の運動会イベント（ランダム発生）
  - テキストで睡眠不足を表現、翌日の仕事への影響を示唆
- 早朝4時の餌催促イベント
  - 選択肢: 「無視して寝る」「起きて餌をやる」
  - 影響: 猫のストレス、プレイヤーの睡眠不足を蓄積
- 発情期の鳴き声イベント（不妊手術していない場合）
  - 近隣からの苦情リスク
  - 選択肢: 「我慢する」「防音対策を検討」「不妊手術を予約」

#### 2-2-3. 衛生面の現実

- 部屋の臭いイベント
  - トイレ掃除をサボると臭いが蓄積
  - 来客イベントで恥ずかしさを体験（テキスト表現）
- 抜け毛の掃除イベント
  - 毎日の掃除機がけ、服に付く毛
- 吐き戻しイベント（ランダム発生）
  - 朝起きたらカーペットに吐き戻し
  - 選択肢: 「掃除する」「後で掃除する」
  - 影響: 臭いの蓄積、衛生面のストレス

#### 2-2-4. 家具・物への被害

- 爪とぎイベント（ランダム発生）
  - 大切なソファが傷つく
  - 選択肢: 「我慢する」「爪とぎグッズを買う（費用発生）」
- 物を壊されるイベント
  - 花瓶、観葉植物、ガラスコップなど
  - 修繕費・買い替え費用が累積

#### 2-2-5. 生活の自由度への影響

- 友人の誘いイベント（ランダム発生）
  - 「今日飲みに行かない？」
  - 選択肢: 「断る」「ペットシッターを頼む（費用発生）」「行く（猫のストレス↑）」
- 旅行計画イベント
  - ペットホテル探しの手間、費用（1泊5,000円〜）
  - 選択肢: 「旅行を諦める」「ペットホテルを予約」

#### 2-2-6. 問題行動のリアル 🔴 最優先

- 粗相イベント（トイレ以外での排泄）
  - ストレスが原因の場合あり
  - 選択肢: 「怒る（逆効果）」「獣医に相談（費用発生）」「環境を見直す」
- 夜鳴きイベント
  - 近隣からの苦情リスク
  - 選択肢: 「我慢する」「防音対策」「原因を調べる」
- 噛みつき・引っかきイベント
  - 遊びの延長、ストレスサイン
  - 選択肢: 「叱る」「遊び方を変える」「獣医に相談」

**成功基準**:

- 各イベントカテゴリを最低1回ずつ体験: 80%以上のユーザー
- 終了時アンケートで「印象に残ったイベント」に上位で挙がる

**決定事項**:

- イベント発生頻度はネガティブ:ポジティブ = 6:4（プレイテストで微調整）
- ゲームオーバーなし、状態が悪化するだけ（最後まで体験させる）

### 2-3. B×Y領域の体験イベント 🟠 MVP推奨

**目的**: 未来の不確実性を認識させる

#### 2-3-1. 想定外の事態（ランダムイベント）

- 飼い主の病気イベント
  - 「インフルエンザで寝込んでしまった」
  - 選択肢: 「家族に頼む」「友人に頼む」「無理して世話する」
- 親の介護イベント
  - 「実家の親が倒れた」
  - 選択肢: 「猫を連れて実家へ」「誰かに預ける」「里親を探す」
- 転勤イベント
  - 「ペット不可物件への引っ越しが決まった」
  - 選択肢: 「ペット可物件を探す（家賃↑）」「転職を検討」「里親を探す」
- 離婚イベント（パートナーがいる場合）
  - 「パートナーと別れることになった」
  - 選択肢: 「自分が引き取る」「相手に渡す」「里親を探す」
- アレルギー発症イベント
  - 「家族が猫アレルギーを発症した」
  - 選択肢: 「アレルギー対策を強化」「別居を検討」「里親を探す」

#### 2-3-2. 健康管理の責任（老齢期）

- タイムスキップで10年後を体験
  - 猫が高齢になり、通院頻度が増加
  - 医療費の累積（月1〜3万円）
  - 介護負担（寝たきり、投薬、点滴）
- 選択肢:
  - 「治療を続ける（費用・時間）」
  - 「緩和ケアに切り替える」
  - 「安楽死を検討」（倫理的選択）

#### 2-3-3. 別れの辛さ

- 最期を看取るイベント（タイムスキップ後）
  - ペットロスの体験
  - 選択肢: 「またすぐに猫を迎える」「しばらく飼わない」「もう飼わない」

**成功基準**:

- ランダムイベントを最低1つ体験: 60%以上のユーザー

**決定事項**:

- MVP期はタイムスキップ機能なし（将来バージョンで追加）
- ランダムイベントの発生頻度はプレイテスト時に調整

### 2-4. 計画外の繁殖イベント 🟡 将来バージョン

**目的**: 不妊手術の重要性を体験させる

**機能内容**:

- 不妊手術の選択イベント（飼育開始時）
  - 選択肢: 「する（費用3万円）」「しない（費用0円）」
  - 影響: 発情期の鳴き声、計画外の繁殖リスク

- 不妊手術をしなかった場合のイベント
  - 脱走した猫が妊娠して帰ってくる
  - 子猫が5匹産まれる
  - 選択肢: 「全頭飼う（経済的・時間的負担5倍）」「里親を探す（手間）」「保護団体に相談」

**実装優先度**: 低（MVP後に検討）

**理由**:

- 本プロダクトにたどり着く時点で、計画外の繁殖をするユーザーは想定しにくい
- 飼育放棄理由の上位ではない
- MVP期はより優先度の高いイベントに注力

### 2-5. A×Z領域の体験イベント 🟡 将来バージョン

**目的**: 猫を幸せにできるかを体験させる

#### 2-5-1. 猫の個性を受け入れられるか

- 期待と現実のギャップイベント
  - 「懐かない猫」「人見知りな猫」
  - 選択肢: 「個性として受け入れる」「懐かせる努力をする」「諦める」

#### 2-5-2. 猫同士の相性（多頭飼い）

- 2匹目を迎えるイベント
  - ケンカイベント
  - 選択肢: 「別々の部屋で飼う」「相性改善の努力」「2匹目を手放す」

#### 2-5-3. 長期的な責任感（覚悟）

- タイムスキップで10年後を体験
  - 「飽きた」イベント
  - 選択肢: 「それでも飼い続ける」「里親を探す」

#### 2-5-4. 家族の同意・協力

- 家族が非協力的なイベント
  - 「パートナーが世話をしてくれない」
  - 選択肢: 「一人で抱え込む」「家族会議を開く」「諦める」

#### 2-5-5. ポジティブな側面（喜び・癒し） 🔴 MVP必須

- 癒しイベント（ランダム発生）
  1. **猫があなたを見つめてくる**（信頼の証）
     - 発生条件: なつき度が一定以上、プレイヤーが猫の近くにいる
     - 効果: プレイヤーの体力回復、ストレス軽減
     - 実装: テキスト表示 + 猫の表情変化（静止画）
  2. **猫が膝に乗ってくる**（愛着の表現）
     - 発生条件: なつき度が高い、プレイヤーが座っている
     - 効果: なつき度微増、プレイヤーの満足度向上
  3. **猫が遊びに誘ってくる**（愛らしい行動）
     - 発生条件: 猫の体力・健康度が高い、時間帯が夕方
     - 効果: 一緒に遊ぶことでなつき度向上、プレイヤーの満足度向上
- 絆が深まる選択肢
  - 「一緒に遊ぶ（時間を使う）」「写真を撮る」「SNSに投稿」

**決定事項**:

- MVP必須に変更（将来バージョンから変更）
- 最低3個のポジティブイベントを実装
- イベント発生頻度: ネガティブ:ポジティブ = 6:4
- 工数: +6時間（各2時間 × 3イベント）

## 3. 金銭管理機能

### 3-1. 飼育費用の可視化 🟠 MVP推奨

**目的**: 経済的負担のリアルを体験させる

**機能内容**:

- **初期費用**
  - グッズ購入: 2〜3万円
  - 不妊手術: 3万円（選択制）
  - ワクチン接種: 5,000円

- **月次固定費**
  - 食費: 5,000〜10,000円/月
  - トイレ砂: 1,000〜2,000円/月
  - その他消耗品: 1,000円/月

- **突発的費用**
  - 病気・ケガの治療費: 5,000〜50,000円/回
  - 爪とぎ・家具の買い替え: 5,000〜30,000円/回
  - ペットホテル: 5,000円/泊

- **累積費用の表示**
  - 総支出額をテキストで表示
  - 予想生涯費用（126万円）との比較

**成功基準**:

- プレイ後アンケートで「経済的負担を実感した」: 80%以上

**決定事項**:

- MVP推奨に変更（MVP必須から変更）
- 経済的負担は基準が明確（A×X領域）で、事前情報提供で対応可能
- ゲーム内では突発的費用の発生をテキストで表現
- グラフ表示は将来バージョンで検討

## 4. 振り返り・フィードバック機能

### 4-1. プレイログの記録 🔴 MVP必須

**目的**: ユーザーの行動を分析し、振り返りに活用

**機能内容**:

- 記録するデータ:
  - イベントごとの選択肢
  - 猫の状態変化（健康度、なつき度、ストレス度）
  - 総支出額
  - プレイ時間
  - 離脱ポイント（途中でやめた場合）

**データ活用**:

- 終了時レポート生成（4-2）
- プレイ後アンケートの質問カスタマイズ
- 開発側の分析（どのイベントで離脱が多いか）

---

### 4-2. 終了時レポート 🔴 MVP必須

**目的**: 気づきを振り返らせる

**機能内容**:

- **飼育適性診断**
  - あなたの飼育スタイル: 「マメなタイプ」「大雑把なタイプ」など
  - 苦手だったこと: 「睡眠不足」「衛生面」など
  - 得意だったこと: 「遊び相手」「スキンシップ」など

- **統計データとの比較**（詳細版）
  - 「あなたが体験した22のシーンは、実際の飼育放棄理由に基づいています」
  - 「睡眠不足で苦労したあなたは、実際の飼い主の43%と同じです」
  - 令和5年度の猫の殺処分数: 6,899頭
  - 主な原因: 飼い主の病気・介護（32%）、転居（10%）、離婚（8%）、問題行動（3%）

- **気づきポイントの明示**
  - 「こういう場面が苦手だった」リスト
  - 「実際はもっと大変」という注意書き
  - 「ゲームで飼えた ≠ 実際に飼える」という警告

**成功基準**:

- レポート閲覧率: 80%以上

**決定事項**:

- 詳細な統計データは終了時レポートで表示
- ユーザーが体験を振り返りながら統計データを参考に判断できる
- MVP期: テキストベース / 将来: グラフ化・インフォグラフィック化

---

### 4-3. プレイ後アンケート 🔴 MVP必須

**目的**: 行動変容を測定

**機能内容**:

- 「プレイ後、猫を飼いたいと思いますか？」
  - 非常に飼いたい / やや飼いたい / もう少し考えたい / やめておこうと思う
- 「プレイ前と比べて、飼育に対する理解は深まりましたか？」
  - 5段階評価
- 「どのシーンが最も印象に残りましたか？」
  - 自由記述
